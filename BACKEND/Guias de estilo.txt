## üß≠ 1. Estructura general y formato

### ‚úÖ **Indentaci√≥n**

* Python **usa indentaci√≥n para definir bloques** (no llaves `{}` como en C o Java).
* **Convenci√≥n:** 4 espacios por nivel (no uses tabuladores).

  ```python

      print("Positivo")
  else:
      print("No positivo")
  ```

---

### ‚úÖ **Longitud de l√≠nea**

* No m√°s de **79 caracteres** por l√≠nea de c√≥digo.
* Para comentarios o docstrings, m√°ximo **72 caracteres**.

---

### ‚úÖ **L√≠neas en blanco**

* 2 l√≠neas en blanco entre definiciones de funciones o clases.
* 1 l√≠nea en blanco entre m√©todos dentro de una clase.

```python

    def hablar(self):
        pass

    pass
```

---

### ‚úÖ **Espacios en expresiones y sentencias**

Evita espacios innecesarios:

‚ùå Malo:

```python

```

‚úÖ Correcto:

```python

    print(x)
```

Otros ejemplos:


## üß© 3. Comentarios y documentaci√≥n

### ‚úÖ **Comentarios**

* Usa `#` para comentarios simples.
* Empieza con may√∫scula y termina sin punto, salvo que sea una oraci√≥n completa.

```python

```

### ‚úÖ **Docstrings**

* Se colocan entre triple comillas `"""` justo debajo de una funci√≥n, clase o m√≥dulo.
* Explican qu√© hace, no c√≥mo lo hace.

```python
def calcular_area(base, altura):
    """Calcula el √°rea de un tri√°ngulo."""
    return (base * altura) / 2
```

### ‚úÖ **Orden recomendado:**

1. Librer√≠as est√°ndar (`os`, `sys`, `datetime`, etc.)

3. M√≥dulos locales del proyecto (`from app.models import Usuario`)

Ejemplo:


* Agrupa importaciones con una l√≠nea en blanco entre cada grupo.
